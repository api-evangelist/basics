API Lifecycle Basics

This is one of my lessons on the basics of an API life cycle, walking through some of the common elements API providers should be considering as they work standardize how they deliver APIs across organizations.

The home page of this interactive project is a subway map which provides a single "line", which walks through the basics of my proposed API life cycle. You can click on any stop below, or just begin by clicking on definitions.

This project runs using Jekyll, which is completely hosting using Github Pages. You can access the Github repository for the project, and find the lifecycle details stored within the underscore lifecycle collection. I am just using Jekyll as a hypermedia client to walk through each stop along the life cycle, with the subway map as a visual aid.

I have expanding versions of this life cycle with thousands of stops, spanning over a hundreds lines, providing a wealth of detail that can be applied to the API life cycle, depending on the goal of an organization. If you are interested in producing a subway map for your API operations, or would like an API governance strategy laid out in this way, let me know.
